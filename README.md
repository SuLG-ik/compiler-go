## Вариант задания

Вариант 74 посвящён распознаванию функции языка Kotlin в рамках ограниченного подмножества лексем. Реализованный лексер ориентирован на конструкцию вида:

```kotlin
fun calc(a: Int, b: Int, c: Int): Int {
    return a + (b * c)
};
```

Поддерживается только тот набор лексем, который задан спроектированным конечным автоматом. Лексер не пытается анализировать весь язык Kotlin целиком.

## Диаграмма состояний

![Диаграмма состояний конечного автомата](docs/lexer_fsm.png)

### Краткое описание работы автомата

- Начальное состояние `0` определяет, к какому классу относится текущий символ.
- Состояние `1` считывает последовательность букв и затем классифицирует её как ключевое слово или идентификатор.
- Состояние `2` объединяет подряд идущие пробельные символы в одну лексему.
- Состояния `12`, `13` и `14` используются для распознавания целых и дробных чисел.
- Операторы и разделители из одного символа распознаются сразу при выходе из начального состояния.
- Если символ не соответствует ни одному допустимому переходу, формируется лексическая ошибка.
